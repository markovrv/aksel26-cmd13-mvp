# ESP32-CAM Tank Robot

Робот-машинка на базе ESP32-CAM (AI-Thinker) с дистанционным управлением через WebSocket и видеотрансляцией MJPEG по Wi-Fi.

Проект использует **Python-прокси-сервер** (aiohttp) в качестве посредника между браузерами и ESP32. Это позволяет:
- Поддерживать множество одновременных зрителей
- Реализовать чат и ИИ-бота (YandexGPT)
- Управлять правами администратора
- Менять IP адрес робота на лету

## Архитектура

```
Браузер (HTML/JS) ←→ Python-прокси (aiohttp, порт 8080) ←→ ESP32-CAM
                                                              ├─ HTTP (порт 80)  — /config
                                                              ├─ MJPEG (порт 81) — видеопоток
                                                              └─ WebSocket (порт 82) — команды движения
```

### Компоненты

| Компонент | Назначение |
|-----------|------------|
| `robot_car/robot_car.ino` | Прошивка ESP32: камера, моторы L298N, HTTP/MJPEG/WebSocket серверы |
| `server/server.py` | Python-прокси (aiohttp): ретрансляция MJPEG, WebSocket-прокси, чат, ИИ-бот |
| `server/www/` | Веб-интерфейс (HTML/CSS/JS) |

### Порты ESP32
| Порт | Протокол | Назначение |
|------|----------|------------|
| 80   | HTTP     | Endpoint `/config` для настроек камеры и мотора |
| 81   | HTTP     | MJPEG видеопоток с камеры (multipart/x-mixed-replace) |
| 82   | WebSocket| Приём команд управления: `{"x":-100..100,"y":-100..100,"t":timestamp}` |

### Порты Python-прокси
| Порт | Протокол | Назначение |
|------|----------|------------|
| 8080 | HTTP     | MJPEG-стрим `/stream`, статика `/`, прокси `/config` |
| 8080 | WebSocket| `/ws` — чат, авторизация, команды управления |

## Возможности

### Управление движением
- **Джойстик** (мобильные устройства): nipplejs → WebSocket → ESP32 → L298N
- **Клавиатура WASD** (ПК): keydown/keyup → плавный разгон → WebSocket → ESP32 → L298N
- **Танковая схема**: left = y + x, right = y - x, нормализация при >100%
- **Чистый поворот**: при |y| < 10% и |x| ≥ 10% — разворот на месте
- **Плавный разгон/торможение** на ESP32 с настраиваемым шагом (accel)

### Чат и ИИ-бот
- Встроенный чат для всех зрителей
- ИИ-бот (YandexGPT) отвечает на вопросы о роботе, управлении и трансляции
- Автоматическое определение вопросов (отсеивание команд управления)
- История диалога в рамках сессии (сбрасывается при отключении всех клиентов)

### Администрирование
- Авторизация по паролю для получения прав администратора
- Только один администратор единовременно
- Смена IP адреса ESP32 через веб-интерфейс
- Панель администратора с настройками

### QR-сканер
- Автоматическое сканирование QR-кодов с видеопотока робота
- Запускается при получении прав администратора
- Распознанный текст отправляется в чат от имени администратора

### Настройки камеры (HTTP /config)
- `accel` — интервал шага разгона (мс)
- `quality` — качество JPEG (4-63)
- `framesize` — разрешение камеры (0-10)
- `brightness` — яркость (-2..2)
- `contrast` — контраст (-2..2)
- `fps` — частота кадров видео (1-30)
- `flash` — LED вспышки (0/1)

### Низкая задержка
- `WiFi.setSleep(false)` — отключение энергосбережения WiFi
- `setNoDelay(true)` — отключение Nagle (TCP_NODELAY) на WebSocket сокетах
- Throttle команд на клиенте (50 мс) — отправка только последнего состояния
- `sendWsCommandImmediate()` — немедленная отправка (для остановки)

### Надёжность
- Авто-реконнект WebSocket (2 сек)
- Остановка моторов при потере соединения WS
- Heartbeat WebSocket (5s ping, 3s timeout, 2 retries)
- Остановка моторов при потере фокуса окна (blur)
- Плавный разгон/торможение на ESP32

## Быстрый старт

### 1. Загрузка прошивки на ESP32
```
Плата: ESP32-CAM AI-Thinker
Среда: Arduino IDE с поддержкой esp32
Зависимости: WebSockets by Markus Sattler
```

1. Откройте `robot_car/robot_car.ino` в Arduino IDE
2. Настройте WiFi SSID и пароль в коде
3. Загрузите на ESP32-CAM

### 2. Запуск Python-прокси
```bash
cd server
pip install -r requirements.txt
python server.py
```

Сервер запустится на `http://0.0.0.0:8080`

### 3. Подключение
1. Откройте в браузере `http://<IP-прокси>:8080`
2. Представьтесь (введите имя)
3. Для управления роботом нажмите «Стать администратором» и введите пароль (по умолчанию: `12345`)
4. При необходимости укажите IP адрес ESP32 в панели администратора

## Конфигурация

### Параметры Python-прокси (`server/server.py`)
| Переменная | Значение по умолчанию | Описание |
|------------|----------------------|----------|
| `ESP32_HOST` | `192.168.0.9` | IP адрес ESP32 |
| `SERVER_PORT` | `8080` | Порт прокси-сервера |
| `ADMIN_PASSWORD` | `12345` | Пароль администратора |
| `LLM_ENABLED` | `True` | Включить ИИ-бота |
| `LLM_BOT_NAME` | `Ассистент` | Имя бота в чате |

### Подключение ESP32
- WiFi: SSID = `11935`, пароль = `9091433506`
- Плата: ESP32-CAM AI-Thinker
- Мотор-драйвер: L298N (IN1=GPIO2, IN2=GPIO14, IN3=GPIO15, IN4=GPIO13)
- Вспышка: GPIO4
